# Инкрементальная маршрутизация в динамически реконфигурируемых графах

[![Python 3.9+](https://img.shields.io/badge/python-3.9+-blue.svg)](https://www.python.org/)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

Код для статьи **«Инкрементальная маршрутизация в динамически реконфигурируемых графах сетей связи и промышленного интернета вещей»**.

## Описание

Реализация инкрементального алгоритма обновления кратчайших путей с механизмом стабилизации (порог η). Код воспроизводит все эксперименты, описанные в статье:

1. Сравнение вычислительной сложности с полным пересчётом Дейкстры и A* (алгоритм A* приведён справочно)
2. Анализ влияния порога η на дрожание маршрутов (Churn)
3. Анализ влияния порога η на качество путей (Stretch)
4. Компромисс «Качество vs Стабильность»
5. Сравнение размера области влияния на Mesh и AMI топологиях
6. Границы применимости при серии последовательных отказов

## Ограничения текущей реализации

- Подбор оптимального порога η выполняется **только на регулярной решётке (Grid)**
- Граница эффективности при массовых отказах (~3,5%) получена **только для регулярной решётки**
- Эксперименты на Mesh и AMI топологиях оценивают **исключительно размер области влияния** при η = 0
- Алгоритм A* используется **справочно** и не является прямым конкурентом в постановке SSSP

## Установка

**Требования к окружению:**
- Python 3.9+
- Протестировано на Python 3.11.7

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/Adamosig/incremental-routing-simulator.git
   cd incremental-routing-simulator
   ```
2. Создайте виртуальное окружение (опционально):
   ```bash
   python -m venv venv
   source venv/bin/activate  # Linux/macOS
   venv\Scripts\activate     # Windows
   ```
4. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
   
## Запуск
   ```bash
   python code_of_article.py
   ```
Скрипт выполнит все 6 экспериментов и сохранит графики в папку ./graphs/.

Ожидаемое время выполнения: от минуты (зависит от производительности компьютера).
